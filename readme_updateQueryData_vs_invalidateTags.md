# 🔄 updateQueryData vs. ♻️ invalidateTags

| Мета            | `updateQueryData`                                             | `invalidateTags`                                                                                            |
| --------------- | ------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| Що робить       | Оновлює локальний кеш RTK Query вручну, без запиту до сервера | Очищає кеш → тригерить автоматичний re-fetch. Позначає певний кеш (асоційований із тегом) як "неактуальний" |
| Мережевий запит | ❌ Не виконується                                             | ✅ Повторно виконує query до API                                                                            |
| Коли зручно     | Дані вже є локально або ти тільки що їх отримав               | Дані могли змінитись на бекенді, треба оновити                                                              |
| Переваги        | Миттєве оновлення інтерфейсу без мережевої затримки           | Дані завжди будуть свіжі                                                                                    |
| Недоліки        | Можеш випадково зробити неконсистентний кеш                   | Зайвий трафік і "дергання" сервера                                                                          |

# ✨ Як правильно вибирати?

| Ситуація                                     | Що краще          |
| -------------------------------------------- | ----------------- |
| Ти маєш повну актуальну гру (game) локально  | `updateQueryData` |
| Ти тільки що зробив PATCH/POST/DELETE        | `invalidateTags`  |
| Потрібно пересмикнути дані з сервера         | `invalidateTags`  |
| Хочеш миттєво оновити інтерфейс без re-fetch | `updateQueryData` |
